# @sommycomfort/backend — Convex

The SommyComfort backend (AR3′/AR4′ — see the **Backend Platform Addendum** in
`_bmad-output/planning-artifacts/architecture.md`). Convex provides the document DB,
reactive queries, transactional mutations, actions (external calls/webhooks), built-in
realtime, auth, file storage, and scheduling — replacing the superseded NestJS/Prisma/
Postgres/Socket.IO/Redis stack.

**Convex project:** `bry-code/sommycomfort` — dev deployment `quixotic-boar-465`,
prod `notable-cod-441`.

## Layout

```
convex/
  schema.ts        # defineSchema — tables added per-story (currently: auditLogs)
  auditLogs.ts     # record (mutation) + listForEntity (query) — AR9
  health.ts        # check (query) — liveness
  _generated/      # generated by `convex dev`/`convex codegen` (NOT committed)
```

## First-time setup (requires Convex login — not possible offline)

```bash
pnpm --filter @sommycomfort/backend dev      # `convex dev`: logs in, links the dev
                                             # deployment, writes .env.local + _generated
```

This is **deferred in the current environment** (no `CONVEX_DEPLOYMENT` / network auth):
`convex codegen` fails with "No CONVEX_DEPLOYMENT set" until someone runs `convex dev`
against `quixotic-boar-465`. Until then:

- `convex/_generated/` does not exist, so `convex/auditLogs.ts` / `health.ts` (which import
  `./_generated/server`) won't type-check in an editor — **expected** until codegen runs.
- This package intentionally has **no `typecheck`/`test` script wired into the workspace
  gates**, so `pnpm typecheck/test/build` stay green. Function type-checking + `convex-test`
  unit tests are added once `convex dev` has produced `_generated`.

## Scripts

- `dev` → `convex dev` (watch + codegen against the dev deployment)
- `codegen` → `convex codegen` (regenerate `_generated`)
- `deploy` → `convex deploy` (ships to prod `notable-cod-441`; gate on `CONVEX_DEPLOY_KEY` in CI)
