# QA Checklist

## Functional QA

- Guest can search rooms.
- Guest can submit booking.
- Staff can create direct booking.
- Reception can check in guest.
- Reception can check out guest.
- Caretaker can update room status.
- Housekeeping can complete task.
- Admin can manage rooms and room types.
- Payments and balances calculate correctly.
- Reports match source transactions.

## PWA QA

- App installs on Android and desktop.
- Offline fallback works.
- Staff task updates queue or block correctly while offline.
- Theme persists after reload.
- Push notification setup is testable.

## Security QA

- Roles cannot access unauthorized modules.
- Guest ID documents are protected.
- Staff permission changes are audited.
- Payment changes are audited.
- Exports respect permissions.

## Visual QA

- Dark mode and light mode are complete.
- Text fits on mobile.
- Buttons have stable dimensions.
- Icons are consistent.
- Tables remain usable on small screens.
